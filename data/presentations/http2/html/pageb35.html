Première étape, se procurer un certificat SSL

  • Comme dirait Séraphin Poudrier, un certificat SSL, ça coûte cher, viande à chien.
  • La solution : Let's encrypt
    • Sur votre serveur Linux, installer "Let's encrypt"
    • Pré-requis : Installer git et bc (yum install git, apt-get install git)
    • Installer dans /opt/letsencrypt sudo git clone https://github.com/letsencrypt/letsencrypt /opt/letsencrypt
    • Arrêter le serveur Web et s'assurer que le port 443 est bien ouvert (voir /etc/sysconfig/iptables)
    • ./letsencrypt-auto certonly --standalone -d test.benoitpiette.com
    • letsencrypt va finaliser son installation et vous demandera la liste des domaines
    • si tout va bien, les certificats et clés privées seront disponibles dans /etc/letsencrypt/live/test.benoitpiette.com
  • Votre DNS doit vraiment pointer vers ce serveur pour que let's encrypt puisse valider que vos domaines vous appartiennent
  • Les certificats seront valides pendant trois mois. Vous pouvez automatiser les renouvellements par des scripts.
  • Plusieurs plugins existent, entre autres pour Apache
